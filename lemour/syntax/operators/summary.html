The descriptions in this table are a bit long, but that's just for the sake of clarity. Key:
  1. For the heading row
    1. Op means Operator
    2. N means Number of (operand) parameters
    3. N means Number of (operand) parameters
  2. = is used mathematically to describe what the pieces code do.
  3. \= is the assignment operator from the lemour language. It is normally just written as =, but it's backslash-escaped in this table for clarify. Backslash itself has various uses in the language and is not described here, because it is not an operator.
Overloadable operators
Op N Description
+ 2 Addition: a + b = (a added to b), (a add b), or (a plus b).
++ 2 Multiplication: a ++ b = (a multiplied by b), or (a times b).
+++ 2 Exponentiation: a +++ b = (a to the power of b), or (a to the b).
- 2 Subtraction: a - b = (a subtracted by b), (b substracted from a), (a subtract b), or (a minus b).
-- 2 Division: a -- b = (a divided by b), (a divide b), or (a over b).
--- 2 Logarithm: a --- b = the base-b logarithm of a = (the natural logarithm of a) divided by (the natural logarithm of b).
--^ 2 Root: a --^ b = the b-th root of a = a to the power of (1 divided by b).
. 2 Currying: a . b = the function a, executed with the (argument or arguments) b already passed into a.
.. 2 Nesting: a .. b = (c(aa:..) := a(b(aa:..))) the function a, executed with the (argument or arguments) b already passed into a.
Trying to overload the following operators causes the compiler to throw a syntax error.
Non-overloadable operators
Op N Description
@ 2 Overloading: the overloading operator cannot be overloaded. Sorry. The left hand side is a list of values to curry in. You can ommit and use this as an unary operator instead. The right hand side is the operator to overload. This must be followed by a function declaration, since that's the whole point of these operators..
